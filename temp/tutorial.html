Understanding the basic

In order to make a website, we have to understand the structure of the website and how they work.
There are 3 necessary components that make up a website, which are:

  • A framework for Frontend
  • A framework for Backend
  • A REST Api

Understanding the Frontend

Frontend is where the clients (users) can interact with the website, whether to ask for information, make bank transaction, download files, etc.


Understanding the Backend

Let's have a look at Backend. Backend is where all the data are stored and protected. In order to store the data, we need to come up with different types of data. For example, a contact in your phone requires certain information, such as name of the contact, phone number, .... We need a MVC model.
MVC stands for Mode - View - Control.
Model is where we narrow down the data to store to the database.
View is the method of viewing the data.
Control is to control special instruction to manipulate and transfer data.


Connecting the pieces

Now that we have the Frontend and Backend, we need to somehow connect them together. That's where the API comes in.
API (Application Programing Interface) is the connection between Frontend and Backend.
When frontend wants to get or post data, it will request the Api for the permission, API passes the request to the Backend and return the response from Backend. When new data is passed from the admin into the Backend database, it will send the information to the API and display it on Backend.


Finally, Format the page

Formating a page does not require a keen eye on art. There are certain concept that you need to keep. Most important aspects are:

  • Make them responsive.
  • Make it clean. This means that the webpages should look clean and clear.
  • Make it user-friendly.
  • Be creative.
